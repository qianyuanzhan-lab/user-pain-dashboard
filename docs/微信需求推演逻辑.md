| 规则 | 示例 | 说明 |

| 长度过短 | "垃圾" "差评" | 长度 ≤15 字 |

## 四、阶段三：需求识别与分类

### 4.1 需求定义字典

预定义 18 个需求类别，每个类别包含：
- `demand`: 需求描述

## 五、阶段四：聚类合并

### 5.1 合并逻辑

同一需求类别下的所有评论合并为一个「需求点」：
- **标题**：需求描述（如"希望能找到人工客服解决问题"）
- **提及数**：命中该类别的评论数量
- **典型样本**：选取 3-5 条最具代表性的评论

### 5.2 样本选择算法

```python
def select_best_samples(reviews, max_samples=5):
    scored = []
    for review in reviews:
        score = 0
        # 1. 内容长度得分（50-200字最佳）
        # 2. 高质量特征得分（功能建议、使用场景）
        # 3. 与标题相关性得分（关键词命中）
        # 4. 语义完整性得分（主谓宾结构）
        scored.append((score, review))
    
    # 按得分降序，取前N个
    return [r for _, r in sorted(scored, reverse=True)[:max_samples]]
```

## 八、核心算法总结

### 8.1 需求识别三步法
1. **关键词匹配**：正则表达式识别需求类别
2. **排除过滤**：剔除误匹配（如"客服不错"不是客服问题）
3. **相关性加权**：至少命中1个相关词才入选

### 8.2 样本质量评估五维度
1. 内容长度（20-200字最佳）
2. 高质量特征（功能建议、场景描述）
3. 与标题相关性（关键词命中率）
4. 语义完整性（主谓宾结构）
5. 情绪适度（纯负面情绪扣分）

### 8.3 三梯队排序原则
- **T1 先排**：功能缺失和明确问题优先处理
- **T2 中间**：具体能力改进作为主体
- **T3 后排**：泛化体验优化延后
